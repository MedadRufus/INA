The packages are small and a lot of work to solder, but fortunately there are now several sources for breakout boards for the various devices which are worth it in time savings. My first test with a I\+N\+A226 involved a blank breakout board, some solder paste, a frying pan, desoldering braid, a magnifying glass and quite a bit of time to set up the first breadboard. \hypertarget{md__r_e_a_d_m_e_autotoc_md14}{}\doxysubsection{Library description}\label{md__r_e_a_d_m_e_autotoc_md14}
The library locates all I\+N\+A2xx devices on the I2C chain. Each unit can be individually configured with 2 setup parameters describing the expected maximum voltage, shunt/resistor values which then set the internal configuration registers is ready to begin accurate measurements. The details of how to setup the library along with all of the publicly available methods can be found on the \href{https://github.com/SV-Zanshin/INA/wiki}{\texttt{ I\+NA wiki pages}}.

Great lengths have been taken to avoid the use of floating point in the library. This keeps the library size down because floating point support doesn\textquotesingle{}t have to be compiled into the code. The results are returned as 32-\/bit integers\+To keep the original level of precision without loss but to allow the full range of voltages and amperes to be returned the amperage .
