Detailed Description

Example program for the INA Library demonstrating reading an INA device and displaying results.

Description

Program to demonstrate the INA library for the Arduino IDE. All INA2xx devices on the I2C bus are located and the program does a simple infinite loop to retrieve and display measurements for the bus voltage and current running through each INA device found.

Detailed documentation can be found on the GitHub Wiki pages at https://github.com/SV-Zanshin/INA/wiki

This example is for a INA set up to measure a 5-Volt load with a 0.1 Ohm resistor in place, this is the same setup that can be found in the Adafruit INA219 breakout board. The complex calibration options are done at runtime using the 2 parameters specified in the "begin()" call and the library has gone to great lengths to avoid the use of floating point to conserve space and minimize runtime. This demo program uses floating point only to convert and display the data conveniently. The INA devices have 15 bits of precision, and even though the current and watt information is returned using 32-bit integers the precision remains the same.

The library supports multiple INA devices and multiple INA device types. The Atmel's EEPROM is used to store the 96 bytes of static information per device using https://www.arduino.cc/en/Reference/EEPROM function calls. Although up to 16 devices could theoretically be present on the I2C bus the actual limit is determined by the available EEPROM - ATmega328 UNO has 1024k so can support up to 10 devices but the ATmega168 only has 512 bytes which limits it to supporting at most 5 INAs. Support has been added for the ESP32 based Arduinos, these use the EEPROM calls differently and need specific code.

void loop ( )

Arduino IDE method called after setup and never exited.

This is the main program for the Arduino IDE, it is called in an infinite loop. The INA devices detected in the setup() call are cycled through and their readings are displayed to the serial port

void setup ( )

Arduino IDE method called once upon device start/restart.

It is only called one time and all of the variables and other initialization calls are done in this method prior to entering the main loop for data measurement.
